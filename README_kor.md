# 🌌 Ansible Station: 메타-에이전트 오케스트레이션 허브

[![Version](https://img.shields.io/badge/Version-3.0.0-blue)](./) [![Last Updated](https://img.shields.io/badge/Last%20Updated-2025--12--30-green)](./)

> *"전송 개시: 세계의 진명(True Names)을 움직이기 시작합니다."*

**Ansible Station**에 오신 것을 환영합니다. 이곳은 **Vibe Coding**을 위한 중앙 지휘 통제소입니다.
이 저장소는 차세대 AI 에이전트(Claude, Gemini 등)를 위한 전역 설정 파일, 스킬 셋, 그리고 진화 프로토콜을 보관합니다. 에이전트 기반 워크플로우를 위한 **표준 운영 절차(SOP)**이자 **지능의 중추(Brain)** 역할을 수행합니다.

---

## 🏛️ 핵심 아키텍처: 3대 기둥 (The 3 Pillars)

이 저장소는 에이전트 시스템의 3단계 진화 과정으로 구성되어 있으며, 기초적인 역할 정의부터 심층적인 인지 사고까지 완전한 프레임워크를 제공합니다.

### 1. 기초: 기본 역할 (`1001`)
📂 **[1001_Agent_Systems_Basic](./1001_Agent_Systems_Basic)**

> *기반(The Foundation).* "나는 누구인가"를 정의합니다.

이 레이어는 AI의 근본적인 정체성을 확립합니다. 범용 LLM을 특정 역할을 수행하는 전문가로 변환합니다.
*   **개념**: 역할 정의 (Definition-centric)
*   **핵심 요소**: 35개 이상의 역할 정의 (아키텍트, 개발자, 보안 가디언 등)
*   **사용 사례**: 특정 작업에 대해 명확하고 단일한 목적을 가진 전문가가 필요할 때.

### 2. 실행 엔진: 자동화 워크플로우 (`1002`)
📂 **[1002_Agent_Systems_Engine](./1002_Agent_Systems_Engine)**

> *엔진(The Engine).* "어떻게 일하는가"를 정의합니다.

이것은 **메인 실행 프레임워크**입니다. 정적인 역할을 넘어 역동적인 행동으로 나아갑니다. **자동 정규식 라우팅(Auto-Routing)**, **실행 가능한 의사결정 트리**, **체이닝 패턴**을 특징으로 합니다.
*   **개념**: 실행 가능성 & 자동화 (Action-centric)
*   **핵심 요소**:
    *   **Auto-Routing**: 정규식 기반으로 적절한 에이전트를 자동 선택하는 디스패처.
    *   **Smart Chain**: `탐색(Discovery)` → `설계(Architect)` → `구현(Implementation)` 패턴.
    *   **AGENTS_LIST.md**: 33명의 전문가 에이전트 전체 명단.
*   **사용 사례**: **(Vibe Coding 권장)** 빠르고 효율적인 자동화 개발 사이클.

### 3. 인지 프레임워크: 심층 사고 (`1003`)
📂 **[1003_Agent_Systems_Thinking](./1003_Agent_Systems_Thinking)**

> *두뇌(The Brain).* "어떻게 사고하는가"를 정의합니다.

이 레이어는 AI의 인지 과정을 업그레이드합니다. 수평적 사고, 문제 재정의, 다차원 분석이 가능한 **"인지 에이전트(Cognitive Agents)"**를 도입합니다.
*   **개념**: 인지 과정 & 철학 (Thought-centric)
*   **핵심 요소**:
    *   **Cognitive Layer**: 통찰 탐험가(Insight Explorer), 다차원 분석가, 문제 재정의가.
    *   **5-Step Thinking Process**: 복잡한 문제 해결을 위한 필수 프로토콜.
*   **사용 사례**: 전략 기획, 정의되지 않은 문제 해결, 창의적 아이디어 도출, R&D.

## 🧩 스킬 모듈 (Skill Modules)

에이전트의 도구함을 강화하기 위해 설계된 특수 능력들입니다.

### **[2001_Skill_skill-generator](./2001_Skill_skill-generator)**
**"설계자 (The Architect)"**
다른 고품질 Claude 에이전트 스킬을 생성하는 메타 스킬입니다. Anthropic의 공식 모범 사례(점진적 정보 공개, YAML Frontmatter)를 준수하여 표준화를 보장합니다.
*   **용도**: 에이전트를 위한 상용 수준의 새로운 스킬 생성.

### **[2001_Skill_translation-specialist](./2001_Skill_translation-specialist)**
**"번역 장인 (The Master Translator)"**
학술적 이론(Nida, ISO 17100)에 기반한 문맥 인식 번역 에이전트입니다.
*   **핵심 기능**: **4계층 다차원 분석** (어휘 → 통사 → 담화 → 화용).
*   **능력**: 사용자 지시 없이도 도메인(법률, 기술, 마케팅)을 자동 감지하고 번역 전략(직역 vs 트랜스크리에이션)을 조정합니다.

## 🧬 진화 트랙 (Evolution Track)

### **[8001_Gemini_Evolution](./8001_Gemini_Evolution)**
**"프로젝트 진화 (Project Evolution)"**
Claude Code의 서브에이전트 아키텍처를 **Gemini Antigravity** 환경으로 이식하기 위한 로드맵이자 기술 가이드입니다.
*   **내용**:
    *   `GEMINI_SUBAGENTS_GUIDE.md`: "페르소나 아티팩트"와 "작업 경계(Task Boundary)"를 Gemini에 접목하기 위한 청사진.
    *   `AGENTS_LIST`를 실행 가능한 워크플로우 파일로 모듈화하는 전략.

## 🚀 사용 방법

1.  **모드 선택**:
    *   **코딩**: `1002_Agent_Systems_Engine`을 사용합니다. `CLAUDE.md`를 프로젝트 루트에 복사하세요.
    *   **기획/사고**: `1003_Agent_Systems_Thinking`을 사용합니다.
2.  **에이전트 활성화**:
    *   `AGENTS_LIST.md`를 참조하여 적절한 전문가(예: `103_security_guardian`)를 찾습니다.
    *   정의된 키워드(정규식)를 사용하여 올바른 에이전트를 자동으로 트리거합니다.
3.  **스킬 배포**:
    *   `2001_Skill_*` 폴더를 로컬 에이전트 설정 디렉토리(예: `.claude/skills` 또는 `.gemini/skills`)로 복사합니다.
